## Стратегия secrets
- Все секреты передаются только через переменные окружения.
- Секреты не хранятся в git-репозитории.
- Для локальной разработки используется `.env` (игнорируется в `.gitignore`).
- Для `stage/prod` секреты задаются через менеджер окружения VPS (systemd unit env, docker secrets или аналог).
