# Техническое задание

## Telegram-бот для управления платным доступом в закрытые Telegram-чаты

## 1. Цель системы

Создать Telegram-бота, который управляет доступом пользователей в закрытые Telegram-чаты по ежемесячной подписке.

Бот должен:
- принимать пользователей по специальным ссылкам (коннекторам);
- собирать данные пользователя (ФИО, телефон, email, username Telegram);
- отправлять пользователя на оплату через ЮKassa;
- после успешной оплаты предоставлять доступ в соответствующий Telegram-чат;
- отслеживать срок подписки;
- удалять пользователя из чата при окончании подписки;
- продлевать доступ при успешном автосписании через ЮKassa;
- отправлять уведомление о продлении;
- предоставлять администратору интерфейс управления.

Система должна заменить текущую связку: Ainox + Robokassa.

## 2. Общая архитектура

Система состоит из следующих компонентов:

1. Telegram-бот (основной интерфейс для пользователей)
2. Backend сервер (логика работы системы)
3. База данных
4. Интеграция с платежной системой ЮKassa
5. Административная панель (админка)

## 3. Концепция «коннектора»

Коннектор — это ссылка подключения пользователя к подписке.

Каждый коннектор содержит:
- привязку к одному Telegram-чату
- стоимость подписки
- срок подписки (1 месяц)

Каждый коннектор генерирует ссылку вида:

`https://t.me/<имя_бота>?start=connector_ID`

  По этой ссылке бот определяет:
  - какой чат подключается
  - стоимость подписки
  - параметры доступа

  ## 4. Логика входа пользователя

  Пользователь переходит по ссылке коннектора.

  Бот определяет ID коннектора и показывает:
  - название подписки
  - стоимость
  - период оплаты (ежемесячно)

  Пользователь должен принять условия:
  - публичной оферты
  - политики обработки персональных данных

  Кнопка:
  **«Принимаю условия»**

  Нажатие кнопки считается акцептом оферты и согласием на обработку персональных данных.

  ## 5. Сбор данных пользователя

  После принятия условий бот последовательно запрашивает:

  1. ФИО
  2. Телефон
  3. Email
  4. Telegram username (если есть)

  Данные сохраняются в базе данных.

  ## 6. Завершение регистрации

  После заполнения данных бот отправляет сообщение:

  > «Спасибо. Ваша заявка оформлена. Осталось оплатить подписку.»

  Появляется кнопка:
  **«Оплатить»**

  ## 7. Оплата

  После нажатия кнопки **«Оплатить»** создается платеж через API ЮKassa.

  Пользователь переходит на платежную страницу ЮKassa и оплачивает подписку.

  После оплаты ЮKassa отправляет webhook на сервер.
  Сервер проверяет статус платежа.

  ## 8. Предоставление доступа в чат

  После успешной оплаты:

  1. Бот пытается автоматически добавить пользователя в Telegram-чат.
  2. Если автоматическое добавление невозможно (ограничения Telegram пользователя), бот отправляет invite link.

  Бот должен быть администратором чата.

  ## 9. Управление сроком подписки

  Подписка действует 30 дней с момента оплаты.

  В базе данных фиксируются:
  - дата оплаты
  - дата окончания подписки
  - статус подписки

  ## 10. Уведомление о продлении

  За несколько дней до окончания подписки бот отправляет пользователю сообщение о необходимости продления.

  Сообщение содержит кнопку **«Оплатить»**.

  ## 11. Автосписание

  Если пользователь подключил автосписание через ЮKassa:
  - система инициирует попытку продления
  - при успешной оплате срок подписки продлевается
  - пользователь остается в чате

  Если автосписание не прошло — применяется обычная логика удаления.

  ## 12. Окончание подписки

  Если оплата не поступила:
  - пользователь удаляется из Telegram-чата

  **Важно:**
  Удаление должно происходить без добавления пользователя в черный список Telegram, чтобы его можно было повторно добавить.

  ## 13. Повторная оплата

  Пользователь может повторно оплатить подписку по той же ссылке коннектора.

  После оплаты доступ в чат восстанавливается.

  ## 14. База данных

  Для каждого пользователя должны храниться:
  - Telegram ID
  - Telegram username
  - ФИО
  - телефон
  - email
  - ID коннектора
  - чат
  - дата оплаты
  - дата окончания подписки
  - статус подписки
  - история платежей

  ## 15. Административная панель

  Администратор должен иметь возможность:

  ### Управление коннекторами
  - создать коннектор
  - указать чат
  - указать цену
  - получить ссылку подключения

  ### Управление пользователями
  - список пользователей
  - статус подписки
  - дата окончания

  ### Управление доступом
  - вручную продлить подписку
  - вручную удалить пользователя
  - вручную добавить пользователя

  ### Аналитика
  - количество активных подписок
  - количество оплат
  - отключенные пользователи
  - статистика по коннекторам

  ## 16. Масштабируемость

  Система должна поддерживать:
  - более 100 коннекторов
  - более 300 пользователей

  ## 17. Сервер

  Система должна размещаться на VPS сервере.

  Backend должен поддерживать:
  - Telegram Bot API
  - Webhook от ЮKassa
  - базу данных (PostgreSQL или аналог)

  ## 18. Результат разработки

  Разработчик должен предоставить:
  - рабочий Telegram-бот
  - backend систему
  - базу данных
  - админ-панель
  - интеграцию с ЮKassa
  - инструкции по установке
  - исходный код проекта
